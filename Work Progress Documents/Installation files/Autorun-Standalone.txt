==========AUTORUN STANDALONE ETRACS APPLICATION ONCE SERVER IS TURNED ON=============

1.) Open Git Bash and login to your server credentials of the standalone etracs

2.) Try running first the etracs platform to check there are no errors

    cd ~/etracs255-server-municipality/bin

    sh run.sh

3.) If the platform in the browser is run successfully, we can now proceed in creating our autorun script, otherwise try to resolve the error. 

    http://[SERVERIP]:8080

4.) Stop the platform by pressing Ctrl + C

5.) Go to the /home directory

    cd

    cd ..

6.) Create a reboot file for etracs

    sudo nano reboot.sh

7.) Copy the script below.

    #!/bin/bash

    #Go to the bin etracs directory
    cd /home/[USER]/etracs255-server-municipality/bin

    #Path to run.sh script
    script_path="/home/[USER]/etracs255-server-municipality/bin/run.sh"

    #Run the script using the source command and load the detailed bootload to a log file
    source "$script_path" >> /home/[USER]/etracs255-server-municipality/etracsjob.txt

    #Indicate the date & time of the bootload to monitor application logs (this is optional)
    echo "Etracs Application was executed at $(date +"%m-%d-%Y %H:%M:%S")" >> /home/[USER]/etracs255-server-municipality/etracslog.txt

    **Note: You can specify the location of your log

8.) Make the file executable (this will highlight the file as green when you run 'ls' command)

    sudo chmod +x reboot.sh

8.) Run the crontab job

    crontab -e

9.) Copy the script below to automatically run the platform once server is rebooted

     @reboot /home/reboot.sh

10.) Create the text files for your log files (etracslog.txt & etracsjob.txt) on the path you specified on step 6 & 9.

11.) Restart the server

     sudo reboot

12.) Check the platform on the browser to verify the etracs platform is now autorun.

     http://[SERVERIP]:8080
